2. Ferramentas Externas

🧠 LLM Wiki (Karpathy)

Base de conhecimento persistente · Fazer AGORA

O que é: Um diretório de markdown interligado (wiki) que eu mantenho entre sessões. Inspirado no método de Andrej Karpathy.

Problema que resolve: Hoje eu releio seus TECs do zero toda sessão. Com o wiki, eu ingiro uma vez e consulto depois — respostas mais rápidas, sem repetir trabalho.

Como usamos: ~/wiki-mentoria-takwara/ com páginas para cada conceito (bambu, PU vegetal, Imperveg, MQTF, Série T...). Você pode abrir no Obsidian e ver o grafo.

Referência original →

📄 docmd.io

Gerador de site de documentação · Testar depois

O que é: Alternativa ao MkDocs. Zero-config, framework-free, produz sites Lighthouse 100 com busca offline e geração automática de llms.txt.

Problema que resolve: MkDocs funciona, mas docmd.io é mais leve, mais rápido de configurar e já gera contexto para IA (llms.txt).

Como testar: Criar um repo separado com npx @docmd/core dev e comparar com o MkDocs atual. Decidir após o programa de 7 dias.

Site oficial →

🔍 Plannotator

Revisão de planos de agentes AI · Baixa prioridade

O que é: Camada de revisão visual para planos e código gerados por agentes AI. Intercepta o passo de aprovação e mostra diff interativo.

Problema que resolve: Se você quiser revisar meus planos antes de eu executar, com edição inline e feedback estruturado de volta pra mim.

Quando usar: Ideal para projetos com código complexo (backend, apps). No seu caso (markdown + documentação), o ganho é menor porque você já me corrige no chat. Reveja após o programa.

4. Fluxo de Trabalho Proposto

🧠 A cada sessão (com LLM Wiki)

  1. Orientação automática — eu leio SCHEMA.md, index.md e log.md para saber o estado atual do conhecimento
  2. Consulto o wiki antes de responder — se o conceito já existe, não preciso re-ler seus TECs
  3. Registro no log — tudo que fizermos fica registrado no log.md do wiki
  4. Atualizo páginas — novos insights viram novas páginas ou atualizações nas existentes

📝 Para criar/editar conteúdo do site

  1. Carrego a skill plan e escrevo um plano markdown
  2. Você aprova ou ajusta
  3. Executo com github-pr-workflow — branch → commits → PR
  4. Você revisa via github-code-review ou diretamente
  5. Merge dispara GitHub Actions → deploy automático

🔁 Sincronia entre repositórios

Você tem pelo menos 3 repositórios com conteúdo relacionado:

  • Mentoria_Tecnologia_Takwara — site da mentoria (MkDocs)
  • Mulheres-Tecem-Amazonia — dossiê BNDES completo (referência científica)
  • plataforma-juventude-solidaria-2026 — projeto social (referência de aplicação)

A skill catalogacao-ecossistema foi feita para isso: mapear DOIs, detectar redundâncias, manter tudo consistente.
