2 Como Eu Funciono

🧠

Memória entre sessões

Eu tenho dois tipos de memória. A memory (perfil do usuário + notas persistentes) carrega automaticamente. O wiki (conhecimento do domínio) eu consulto sob demanda.

🔧

Ferramentas

Posso ler arquivos, escrever, pesquisar na web, executar código, navegar no navegador, gerar imagens, e delegar tarefas para subagentes. Cada ferramenta tem um propósito.

📚

Skills

Skills são procedimentos especializados que eu carrego. Ex: llm-wiki (gerenciar wiki), plan (criar planos), github-pr-workflow (fluxo git).

🗣️

Idioma

Respondo em Português (seu idioma preferido). Se você falar em português, respondo em português. Se quiser em inglês, é só pedir.

Minhas limitações honestas

  • Não tenho acesso ao seu .env — arquivos de configuração do sistema são protegidos
  • Não persisto estado entre sessões exceto via memory/wiki — se não salvamos, não existe
  • Preciso de instruções claras — "faça X" funciona melhor que "o que você acha de X?"
  • Não minto sobre resultados — se algo falhar, eu reporto a falha honestamente
⚡ Como acelerar respostas

Quanto mais específico você for, mais rápido e preciso eu serei. Em vez de "melhora isso", prefira "adiciona uma tabela de espécies com dados de CO₂ no TEC_01". Em vez de "faz uma análise", prefira "analisa o arquivo X e me dá 3 recomendações priorizadas".

Produtividade na oficina: dicas

  • Mãos-livres: Suporte de celular fixo na bancada. Aperta "gravar" e faz. Sem edição, sem takes.
  • Bloco único: Cada vídeo cobre UM tópico (ex: "como furar o nó do bambu", "proporção A+B da Bio-Solda").
  • Aproveitar o time-lapse: Deixe a câmera gravando enquanto trabalha. 1h de oficina vira 30 segundos de time-lapse.
  • Não refaça: Se o vídeo ficou imperfeito, eu edito. Você não precisa ser youtuber — precisa ser o Fábio.
  • Limite semanal: 2-3 sessões de gravação de 20-30 min cada. O resto é comigo.

9 Melhores Práticas

🎯 Seja específico

"Cria uma tabela comparativa entre epóxi e PU Vegetal" funciona melhor que "melhora o conteúdo". Eu preciso de direção clara.

📌 Corrija à vontade

Se eu errar o tom, o formato, ou a direção, me corrija. Não repito o mesmo erro se você apontar uma vez. Guardo no memory.

📋 Prefira passos atômicos

Uma tarefa de cada vez. "Faz A, depois B, depois C" é mais produtivo que "faz A, B e C" — cada passo renderiza melhor.

🔄 Confie, mas verifique

Eu relato resultados honestamente, sempre. Se algo falhou, eu digo. Se algo deu certo, mostro a evidência.

🧠 Use o wiki

Toda vez que você me pergunta algo que já respondi antes, eu consulto o wiki. Isso fica mais rápido com o tempo.

📅 Ritmo semanal

Os 7 passos são o esqueleto. Um passo por dia (ou a cada 2 dias) é o ritmo ideal. Não pule etapas.

O que NÃO fazer

  • Não me peça para hackear/contornar sistemas — eu vou me recusar
  • Não espere que eu lembre de tudo sem o wiki — se não está no wiki, não existe entre sessões
  • Não me deixe sem direção — "o que você sugere?" é bom, "faz X" é melhor

10.2 Terminal vs. Aba vs. Agente — O Mapa Completo

Esta é a maior fonte de confusão. Vamos organizar:

ConceitoO que éExemplo

11.3 Integrações Google e YouTube

Você perguntou como funciona cada integração. Vou detalhar o que é possível hoje e o que depende de configuração.

📸 Google Photos

PerguntaResposta
Posso conectar mais de uma conta? Sim, mas cada conta exige seu próprio fluxo de autorização OAuth (login + permissão). Na prática, sugiro escolher a conta principal que tem as fotos dos protótipos e começar por ela.
Como funciona? Com a chave Google API que você já adicionou, eu posso acessar a API Google Photos se você configurar as permissões OAuth. Sem isso, o caminho mais prático é: você compartilha um link de álbum específico ou faz download das imagens que quer usar.
O que eu (Hermes) faço com as fotos? Recebo as imagens, organizo por módulo da mentoria, redimensiono se necessário, e referencio nos TECs e páginas do site.

☁️ Google Drive

PerguntaResposta
Pode ser acessado? Sim, a API do Google Drive segue o mesmo princípio da API Photos. Com OAuth configurado, eu poderia listar arquivos, baixar PDFs, e organizar pastas. Sem OAuth, o caminho prático é você compartilhar links de arquivos ou pastas específicas.
Vale a pena configurar? Para uso pontual (algumas imagens por semana), não — links manuais são mais rápidos. Para uso intensivo (centenas de fotos para catalogar), sim — a automação via API economiza horas.

▶️ YouTube

PerguntaResposta
Consigo acessar apenas com link? Sim. Com um link do YouTube, eu consigo: extrair a transcrição/legendas, obter metadados (título, descrição, duração), e analisar o conteúdo. A ferramenta web_extract da acesso ao texto da página.
Como seriam as edições? Eu sugiro cortes e edições (ex: "aos 3min15s até 5min40s tem a demonstração do forno — corta isso pra um vídeo separado"), mas não executo a edição de vídeo propriamente dita (não tenho ferramentas de edição de vídeo). A edição final é feita por você ou por uma ferramenta externa.
Gerenciamento de playlists? Com a API do YouTube configurada (OAuth), eu poderia criar playlists, adicionar/remover vídeos, e organizar o canal da mentoria. Sem API, funciona com links manuais.
📋 Resumo prático das integrações

Para começar agora (sem configurar OAuth): você me envia links de fotos, vídeos e documentos que quer usar. Eu processo e incorporo no conteúdo.

Para automatizar depois (com OAuth): configuramos a API do Google e eu acesso direto. Faz sentido quando o volume justificar.

12 Organização Pessoal e Segurança

Você perguntou se é seguro pedir reorganização de diretórios, varredura de PDFs e catalogação. A resposta curta: sim, é seguro — com alguns cuidados.

12.1 Reorganização de Diretórios

OperaçãoSeguro?Por quê
Mover arquivos entre pastas ✅ Sim Git detecta a movimentação. O histórico não se perde.
Renomear arquivos ✅ Sim Git detecta rename. Só tomar cuidado com links internos (atualizo junto).
Criar nova estrutura de pastas ✅ Sim Git rastreia tudo. Se algo quebrar, git revert volta atrás.
Excluir arquivos definitivamente ⚠️ Cuidado Se for commitado, o histórico mantém. Seguro se eu confirmar com você antes.
🛡️ Regra de ouro da reorganização

Nada é perdido para sempre no Git. Se um arquivo foi commitado alguma vez, o histórico preserva. Mesmo após um git rm, o commit original continua existindo. A única coisa irreversível seria apagar o repositório inteiro e criar outro.

12.2 Varredura de PDFs e Catalogação

Você tem PDFs espalhados no vault Obsidian e talvez em outras pastas. Eu posso:

  1. Varrer todos os PDFs em diretórios que você indicar
  2. Extrair metadados (título, autor, ano, páginas) usando a skill ocr-and-documents
  3. Extrair texto com pymupdf (rápido, para PDFs digitais) ou marker-pdf (OCR, para PDFs escaneados)
  4. Catalogar em uma Base de Referências no formato:

12.3 Quando pedir reorganização

Peça sempre que sentir que a estrutura atual atrapalha — arquivos em lugar errado, pastas mal nomeadas, conteúdo misturado. Eu:

  2. Você aprova
  3. Executo em um commit separado (fácil de reverter se necessário)

14.5 Publicar no GitHub e Vercel

# 1. Criar o repositório local
git init
git add -A
git commit -m "feat: primeiro commit"

# 2. Renomear branch (Vercel usa main)
git branch -M main

# 3. Criar repositório no GitHub e conectar
gh repo create seu-usuario/nome-do-repo --public
git remote add origin https://github.com/seu-usuario/nome-do-repo.git
git push -u origin main

14.6 Deploy automático via Vercel

  1. Acesse vercel.com e faça login com GitHub
  2. Clique em "Add New Project"
  3. Escolha o repositório que você acabou de criar
  4. Vercel detecta automaticamente o framework (Next.js, Nextra, etc.)
  5. Clique em "Deploy" — aguarde ~2 minutos

Pronto! Seu site está no ar em https://nome-do-repo.vercel.app

14.7 Atualizar o site

Toda vez que você der git push origin main, o Vercel faz deploy automático. Não precisa buildar manualmente.

14.8 Domínio personalizado (opcional)

No dashboard do Vercel: Settings → Domains → Add. Siga as instruções de DNS.

14.9 Aplicar em outro repositório existente

Para publicar qualquer outro projeto seu no Vercel:

  1. O projeto precisa estar num repositório do GitHub
  2. Vai em vercel.com → Add New Project → seleciona o repo
  3. Se for MkDocs: Vercel detecta como "Other", funciona igual
  4. Se for HTML estático: detecta automaticamente

Dica: O Vercel tem plano gratuito generoso. Você pode publicar quantos sites quiser.
